# Workflow

Use the repository first. Bring in TradingView MCP only when the task actually needs chart context.

## Session start checklist

1. Is this a chart task?
2. Do I actually need TradingView context?
3. Is TradingView connected?
4. If not a chart task, continue without MCP.

## When not to use TradingView MCP

Do not use TradingView MCP for note cleanup, file organization, documentation editing, or generic writing.

If TradingView context does not change the answer, skip MCP and work from the repo files directly.

## When to use TradingView MCP

If TradingView MCP is relevant and available, use it for:
- chart context that matters to the analysis
- Pine drafting or validation
- replay review
- screenshots that support the research record

Use current chart context only when needed. Do not treat an open chart as a reason to use MCP automatically.
